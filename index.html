3D Model Guidelines for Real-time Applications

Follow these guidelines to ensure your 3D models are optimized for real-time applications:

1. Polycount and Geometry

  • Aim for low poly models, typically under 10,000 triangles for main characters/objects
  • Use quads whenever possible for easier editing and cleaner topology
  • Avoid n-gons (polygons with more than 4 sides)
  • Utilize LODs (Level of Detail) for distant objects

2. UV Mapping

  • Create clean, non-overlapping UV maps
  • Maximize texture space usage
  • Use texture atlasing for efficiency
  • Keep UVs within 0-1 space

3. Textures

  • Use power-of-two texture sizes (e.g., 512x512, 1024x1024)
  • Compress textures appropriately (e.g., DXT for normal maps)
  • Create texture sets: Albedo, Normal, Roughness, Metallic, etc.
  • Aim for 2K textures max for main assets, 1K or lower for secondary assets

4. Materials

  • Use PBR (Physically Based Rendering) materials when possible
  • Limit the number of materials per model
  • Avoid highly reflective or transparent materials when possible

5. Rigging and Animation

  • Keep bone count low, typically under 50-70 bones for main characters
  • Use weight painting to ensure smooth deformations
  • Provide clear naming conventions for bones and animations

6. File Formats and Delivery

  • Use industry-standard formats like FBX or glTF
  • Include separate files for high-poly source and game-ready low-poly models
  • Provide documentation for any special instructions or notes
Pro Tip: Always test your models in the target engine or application to ensure they perform as expected in real-time environments.

Pre-Delivery Checklist

Use this checklist to ensure your 3D model is ready for delivery:

  • Polycount is within specified limits
  • All geometry is quad-based (or triangulated if required)
  • No n-gons present in the mesh
  • UV maps are clean and non-overlapping
  • All UVs are within 0-1 space
  • All textures are applied and mapped correctly
  • Texture resolutions match project requirements
  • All normals are facing the correct direction
  • No holes or gaps in the mesh
  • No floating or disconnected vertices
  • Smooth shading is applied correctly
  • Materials are set up correctly (PBR if applicable)
  • Rigging is complete and weights are painted (if animated)
  • Animations play correctly without glitches
  • LODs are created and functioning properly (if required)
  • Pivot point is placed correctly
  • Scale is correct according to project requirements
  • All objects and materials are named according to conventions
  • Model has been tested in the target engine/application
  • All necessary maps (normal, specular, etc.) are included
  • File is saved in the correct format (FBX, glTF, etc.)
  • Documentation is prepared (if necessary)
