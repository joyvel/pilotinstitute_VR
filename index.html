3D assets Guidelines for Real-time Applications - Pilot Institute

Follow these guidelines to ensure your 3D models are optimized for real-time applications and keep pipeline flowing:

1. Polycount and Geometry

  • Aim for low poly models, delete geometry that won't be visible to the camera
  • Use quads whenever possible for easier editing and cleaner topology
  • Add a triangulate modifier to the mesh but only apply it on the export settings just to check how triangles look in complex meshes (optional)
  • Avoid n-gons when possible, although for completely flat surfaces they're ok, just keep the triangle count low in these areas

2. UV Mapping

  • Create clean, non-overlapping UV maps except when several objects use the same texture and color (atlas)
  • Use texture atlasing for efficiency
  • Maximize texture space usage

3. Textures

  • Use power-of-two texture sizes (e.g., 512x512, 1024x1024)
  • Create texture sets: Albedo, Normal, Roughness, Metallic, etc.
  • Aim for 4K textures max for main assets and big parts, follow instructions on texture size which may vary on different objects

4. Materials

  • Use PBR (Physically Based Rendering) materials when possible, or follow instructions from developer
  • Limit the number of texture maps per model
  • Separate all objects that need to have a glass material and parent them to their logical parent object

5. Rigging and Animation (if applicable)

  • Always set up a root bone
  • Provide clear naming conventions for bones and animations
  • Include a text or video explaining the functioning of the rig, drivers used, etc.

6. File Formats and Delivery

  • Please share the blender file with packed textures or the textures on a separate folder
  • According to the developer's specifications export textures required for the game engine
  • Include a separate files for the high poly or original file, and the final low poly models
  • Export .FBX files ready to be imported in Unity or the filetype specified by the developers
  • Provide documentation for any special instructions or notes
Pro Tip: If possible test your models in the target engine or application to ensure they perform as expected in real-time environments.

Pre-Delivery Checklist

Use this checklist to ensure your 3D model is ready for delivery:

  • Poly-count is within specified limits
  • All geometry is quad-based (with a optional triangulate modifier)
  • No n-gons present in the mesh unless for completely flat surfaces
  • UV maps are clean and non-overlapping (or overlapping correctly if creating an atlas)
  • All textures are applied and mapped correctly
  • Texture resolutions match project requirements
  • All necessary maps (normal, specular, etc.) are included, please check all maps for imperfections
  • All normals are facing the correct direction
  • No holes or gaps in the mesh revealing flipped normals underneath
  • Make sure no overlapping vertices or meshes are present
  • No floating or disconnected vertices
  • Smooth shading is applied correctly and sharp edges
  • Rigging is complete and weights are painted on all vertices even if the weight is zero (if animated)
  • Animations play correctly without glitches
  • Pivot points are placed correctly (specially for control surfaces)
  • Scale and rotation is correct according to project requirements (apply transforms)
  • All objects and materials are named according to conventions (parts of the model you can name, otherwise use generic names)
  • Model has been tested in the target engine/application (optional)
  • File is saved in the correct format (FBX, glTF, etc.)
  • Documentation is prepared (if necessary)
