Animation Workflow Guidelines - Pilot Institute

Follow these steps to ensure a smooth animation workflow in Blender:

  1. Set the frame rate to 30 fps in the Output Properties panel.
  2. Set the resolution to 3840 x 2160 pixels (4K UHD) in the Output Properties panel.
  3. Make sure whe you send the file to include all dependencies, please don't pack them in the .blend file
  4. Verify that the scale of models and the world is correct. Apply scale transformations if necessary.
  5. Unpack textures to a relative folder next to the project file. Ensure no textures are missing before sending the file.
  6. Create animations following the provided script, instructions, or audio track for accurate timing.
  7. Render a preview using the viewport or Workbench renderer for approval. Send this preview along with the Blender file.
  8. For each sequence, create a new file using the following naming convention:
    • Original file: helicopter.blend
    • Sequence 4: helicopter_S4.blend
    • Sequence 4, Variation 2: helicopter_S4V2.blend
Pro Tip: For every new Blender file with new animations or variations of the same model, send only the new textures added to the new sequences, these are going to be combined with the other textures in the same textures folder, this is to minimize project size and avoid repeated assets.

Pre-Delivery Checklist

  • Frame rate set to 30 fps
  • Resolution set to 3840 x 2160 pixels (4K UHD)
  • Model and world scales are correct
  • Textures unpacked and all present
  • Animations timed correctly to script/audio
  • Preview render completed
  • File named correctly (e.g., helicopter_S4V2.blend)
  • Include a .txt file for comments or documentation if necessary
  • All necessary files included (Blender file, textures, preview)
